## Tutorial

CAMEL-AI supports multiple installation methods. Start with **pip** for a quick setup, or explore **Docker** and **source** installations for advanced workflows.

<Steps>
  <Step title="Install CAMEL-AI via pip (recommended)">
    ```bash
    pip install camel-ai
    ```
    
    <Note>
      Add extras as needed, e.g.:
      ```bash
      pip install "camel-ai[all,rag,web_tools]"
      ```
    </Note>
  </Step>

  <Step title="Optional: Install via uv (alternate method)">
    **First, install uv**
    
    - <b>macOS/Linux</b>:<br/>
      ```shell
      curl -LsSf https://astral.sh/uv/install.sh | sh
      ```
      <details>
        <summary>Use wget if curl unavailable</summary>
        ```shell
        wget -qO- https://astral.sh/uv/install.sh | sh
        ```
      </details>

    - <b>Windows</b>:<br/>
      ```powershell
      powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
      ```

    <Note>
      Reference [UV installation guide](https://docs.astral.sh/uv/getting-started/installation/).
    </Note>

    **Then install CAMEL-AI using uv**
    ```shell
    uv tool install camel-ai
    ```
  </Step>

  <Step title="Other Methods">
    - **Docker**: see [Docker guide](https://github.com/camel-ai/camel/blob/master/.container/README.md)

    - **From Source (with uv)**:
      1. Clone the repository:
         ```shell
         git clone https://github.com/camel-ai/camel.git
         cd camel
         ```
      2. Install `uv` if needed:
         ```shell
         pip install uv
         ```
      3. Create and activate a virtual environment (Python 3.10â€“3.12):
         ```shell
         uv venv .venv --python=3.10
         # macOS/Linux
         source .venv/bin/activate
         # Windows
         .venv\Scripts\activate
         ```
      4. Install CAMEL-AI with all dependencies:
         ```shell
         uv pip install -e "[all, dev, docs]"
         ```
      5. (Optional) Developer setup:
         ```shell
         uv pip install pre-commit mypy
         pre-commit install
         ```
      6. When finished, deactivate:
         ```shell
         deactivate
         ```
  </Step>
</Steps>
